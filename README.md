# Verse Flavor

An experimental implementation of a Verse-inspired programming language written in Rust.

## Disclaimer

> This project is **not an official implementation** of Verse and is **not affiliated with Epic Games**.
>
> The long-term goal of this project is to be as compatible with Verse as reasonably possible using publicly available information. However, compatibility is not guaranteed.

The implementation is primarily based on the public **Book of Verse**:

* https://verselang.github.io/book/

While the book is an excellent learning resource, it is not a complete language specification. As a result, some language features, behaviors, and edge cases may differ from official Verse implementations.

Where documentation is incomplete or ambiguous, this project may make independent implementation decisions.

## Goals

* Implement a language that closely resembles Verse.
* Follow publicly documented Verse syntax and semantics whenever possible.
* Learn and explore language implementation techniques.
* Build a clean and maintainable interpreter/runtime in Rust.

## Non-Goals

This project does not aim to:

* Replace official Verse tooling.
* Serve as a reference implementation of Verse.
* Guarantee behavioral compatibility with current or future Verse releases.

## Compatibility

Compatibility with Verse is a primary goal of this project.

However, several challenges exist:

* Public documentation is incomplete.
* Some language behavior is undocumented.
* Official implementations may change over time.
* Runtime systems used by official Verse environments are not publicly available.
* Certain advanced language features may require interpretation or approximation.

Because of these limitations, the project should currently be viewed as a **best-effort compatibility implementation** rather than a fully compatible Verse implementation.

## Status

This project is experimental and under active development.

Features may be incomplete, incorrect, or subject to change as additional information about Verse becomes available.

Contributions, bug reports, compatibility reports, and corrections are welcome.
